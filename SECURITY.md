# Security

This project is local-only and intentionally avoids account tools, network calls, deployments, and external APIs.

Do not put secrets, credentials, private traces, customer data, proprietary prompts, or private workspace paths in public receipts. Use synthetic examples when creating public documentation or demonstrations.

If you find a security issue in the scripts or schema, document it locally first and review it with a human maintainer before publishing details. A receipt is a documentation aid, not a substitute for security review, legal review, or authorization controls.
